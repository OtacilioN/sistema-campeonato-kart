# Coding Standards

- Use TypeScript with strict types.
- Keep components small and readable.
- Build mobile first.
- Prefer simple domain services over over-engineered abstractions.
- Keep secrets server-only and out of git.
- Use clear domain names aligned with `CONTEXT.md`.
- Keep app code in `src/`, Prisma in `prisma/`, and references in `references/`.
- Run lint, build and Prisma generation before closing a coding task.
